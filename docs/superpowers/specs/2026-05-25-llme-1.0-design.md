# LLMe 1.0 Design

**Status:** Draft → for implementation
**Date:** 2026-05-25
**Owner:** breed

## Summary

LLMe is a Rust CLI + shell-hook coordinator that applies AI-coding-agent
configuration (rules, skills, plugins, MCP servers, hooks) to the current
shell context based on declarative scope rules — "direnv for your agents."
It materializes a per-scope merged config directory and points each
supported agent at it via the agent's native env var (e.g.
`CLAUDE_CONFIG_DIR`).

## Goals (1.0)

- Single Rust binary `llme` installable via `cargo install` and a Homebrew tap.
- Declarative scope model with four scope kinds: `network`, `host`, `user`, `project`.
- Tags map config bundles to scopes; layered merge across scopes.
- Shell hook integration for zsh and bash (direnv-style).
- Materialized cache dir per scope-hash; symlink/file merge from layered bundles.
- Claude Code adapter (reference implementation). Per-agent adapter trait
  in place so Codex/Crush/OpenCode can be added post-1.0 without architectural
  change.
- ICM MCP integration: auto-inject the `icm` MCP into materialized config; support
  scoped server topology (one host runs `mcp-proxy` + ICM; other hosts connect
  over HTTP); wire agent hooks to call ICM via MCP rather than local CLI.
- GitHub sync: throttled background `git pull --ff-only` on hook tick; explicit
  `llme sync` for push.
- CLI: `init`, `hook <shell>`, `status`, `export`, `sync`, `bundle ls/add`,
  `scope ls`, `tag ls`, `doctor`.

## Non-Goals (1.0)

- Codex / Crush / OpenCode adapters (trait exists, implementations post-1.0).
- Fish/PowerShell support.
- Windows support.
- Secret management — bundles are plain files; users handle secrets out-of-band
  (sops/age/gpg).
- Telemetry (see Appendix A).
- GUI/TUI.
- Conflict resolution UX for git sync (fast-forward only; conflicts are surfaced as errors).

## Architecture

### Components

```
┌─────────────────────────────────────────────────────────────┐
│  llme CLI (single Rust binary, async tokio runtime)         │
├─────────────────────────────────────────────────────────────┤
│  Subsystems:                                                │
│   • config        — parse llmenv.toml, validate              │
│   • scope         — evaluate matchers, compute active set    │
│   • merge         — layered merge of bundles → manifest      │
│   • materialize   — write merged tree to cache dir           │
│   • adapter       — per-agent adapter trait + Claude Code    │
│   • mcp           — ICM MCP injection + server topology      │
│   • hook          — shell-hook codegen (zsh/bash) + runtime  │
│   • sync          — git fetch/pull/push, throttle            │
│   • doctor        — diagnostics                              │
│   • cli           — clap subcommands                         │
└─────────────────────────────────────────────────────────────┘
```

### Config repo layout

User-owned git repo (default `~/.config/llmenv`):

```
llmenv.toml             # scopes, tags, bundle bindings, ICM topology
bundles/
  <bundle-name>/
    AGENTS.md           # rules
    skills/...
    plugins/...
    hooks/...
    mcp.toml            # bundle-local MCP additions (optional)
```

### `llmenv.toml` schema (sketch)

```toml
[settings]
cache_dir = "~/.cache/llmenv"           # default
sync_interval_minutes = 15
default_shell = "zsh"

# Scopes — declarative matchers
[[scope.network]]
id = "home-lan"
match = { gateway_mac = "aa:bb:cc:dd:ee:ff" }   # or ssid / cidr
tags = ["home", "trusted"]

[[scope.host]]
id = "fixed"
match = { hostname = "fixed" }
tags = ["host-fixed", "icm-server"]

[[scope.host]]
id = "still"
match = { hostname = "still" }
tags = ["host-still"]

[[scope.user]]
id = "breed"
match = { user = "breed" }
tags = ["user-breed"]

[[scope.project]]
id = "phaedrus"
match = { path_prefix = "~/git/phaedrus" }
tags = ["phaedrus", "rust"]

# Bundles bound to tags
[[bundle]]
name = "base"
tags = ["user-breed"]                   # any matching scope w/ this tag pulls bundle "base"

[[bundle]]
name = "rust-defaults"
tags = ["rust"]

# ICM MCP topology
[icm]
# On scopes carrying "icm-server", run `mcp-proxy` exposing ICM over HTTP.
server_tag = "icm-server"
server_bind = "0.0.0.0:7878"
# Other hosts in matching networks connect via this URL.
client_url = "http://fixed.local:7878"
# Topic naming convention; {scope} and {project} are substituted.
default_topics = ["context-{project}", "preferences", "decisions-{project}"]
```

### Scope precedence & merge

1. Active scopes = all scopes whose matchers evaluate true at hook tick.
2. Precedence (low → high): `network → host → user → project`.
3. Effective tag set = union of tags from active scopes.
4. Bundle set = all bundles bound to any tag in the effective set.
5. Bundles applied in **scope precedence order**; within a single scope, in
   declaration order.
6. File-level merge: later layers' files overwrite earlier; directories union;
   `AGENTS.md` files concatenate with a separator + provenance header.
7. MCP server lists union; conflicts on the same server name resolved by the
   later layer (with a warning to stderr).

### Materialization

- Hash inputs: ordered list of `(bundle_id, content_hash)` + adapter version.
- Cache path: `<cache_dir>/<adapter>/<hash>/`.
- Build steps:
  1. If cache dir exists → reuse; skip materialization.
  2. Else: stage to `<hash>.tmp/`, then atomic rename.
  3. Per-adapter writer emits agent-native layout under the cache dir
     (e.g. Claude Code: `skills/`, `plugins/`, `mcp.json`, `CLAUDE.md`,
     `hooks/`).
- GC: `llme doctor --gc` prunes cache entries not used in last N days.

### Shell hook

`eval "$(llme hook zsh)"` installs a `precmd` hook:

```sh
__llme_hook() {
  eval "$(LLME_SHELL=zsh llme export)"
}
precmd_functions+=(__llme_hook)
```

`llme export` prints `export KEY=val` lines for the active scope:

- `CLAUDE_CONFIG_DIR` → cache path
- `LLME_ACTIVE_SCOPES` → comma-separated scope IDs (for debugging)
- `LLME_CACHE_DIR` → root cache
- Per-adapter env (Codex/Crush/OpenCode envs reserved, not emitted in 1.0)

Plus side-effects:

- Trigger background `git pull --ff-only` if throttle window elapsed.
- Trigger ICM MCP server lifecycle (`mcp-proxy` start) if active scope has
  the server tag and the proxy isn't running.

### Adapter trait

```rust
trait AgentAdapter {
    fn name(&self) -> &'static str;
    fn env_vars(&self, cache_dir: &Path) -> Vec<(String, String)>;
    fn materialize(&self, manifest: &MergedManifest, out: &Path) -> Result<()>;
}
```

1.0 ships `ClaudeCodeAdapter`. The trait is the seam for future adapters.

### ICM MCP integration

- **Server topology:** scopes can carry the `icm-server` tag (configurable
  name). When active on the local host, LLMe ensures `mcp-proxy` is running
  with the `icm` stdio server behind it, bound to `icm.server_bind`.
  Lifecycle is managed via a pidfile under `~/.local/state/llmenv/`.
- **Clients:** on hosts without the server tag, the materialized
  `mcp.json` includes an HTTP MCP client pointing at `icm.client_url`.
- **Hooks via MCP:** for agents that support hook configuration (Claude Code),
  LLMe-generated hooks invoke ICM via HTTP MCP (e.g. via `mcp-proxy` client
  or a small `llme icm` shim) instead of spawning local CLI processes —
  making hooks portable across the network.
- **Topic substitution:** materialization substitutes `{project}`, `{scope}`,
  `{user}` into the default topic list emitted as a CLAUDE.md hint.

### Sync

- Repo discovered via `settings.config_repo` or `~/.config/llmenv` default.
- On hook tick: if `now - last_pull > sync_interval_minutes`, spawn detached
  `git fetch && git pull --ff-only`. Result cached in
  `~/.local/state/llmenv/sync.json`.
- `llme sync` runs `git add -A && git commit -m "..." && git push` (commit
  message templated from staged file list). Fails loudly on conflicts.

### Errors & diagnostics

- All user-facing errors include: operation, scope context, suggested fix.
- `llme doctor` checks: config parses, all scopes evaluable, cache writable,
  git remote reachable, ICM MCP server reachable (if client), shell hook
  installed.

## Testing strategy

- **Unit:** scope matcher evaluation, layered merge, hash determinism,
  adapter output snapshot tests, TOML round-trip.
- **Integration:** drive the binary with a temp `HOME`, fixture configs,
  assert materialized cache contents byte-for-byte.
- **Property-based:** merge associativity within a precedence layer; hash
  stability under bundle reorder when bundles don't conflict.
- **No mutation testing or coverage gates for 1.0** — add post-1.0 once
  the surface stabilizes.

## CLI surface

| Command | Purpose |
| --- | --- |
| `llme init [--repo URL]` | Scaffold or clone config repo, write starter `llmenv.toml`. |
| `llme hook <shell>` | Emit shell-hook source code. |
| `llme status` | Show active scopes, tags, bundles, cache path. |
| `llme export` | Emit `export KEY=val` lines for the current scope. |
| `llme sync` | Commit + push pending changes; also force a pull. |
| `llme bundle ls` / `add <name>` | List / scaffold bundles. |
| `llme scope ls` | List defined scopes and which are active. |
| `llme tag ls` | List tags and bound bundles. |
| `llme doctor [--gc]` | Diagnostics; optional cache GC. |

## Platform / runtime

- macOS (arm64, x86_64) + Linux (x86_64, arm64).
- Rust 1.95+ stable, edition 2024; `tokio` 1.52 runtime; `clap` 4.6 for CLI; `serde` 1 + `toml` 1.1; `git2` or
  shelling out to `git` (decide during impl — leaning shell-out for simplicity).
- Locked-down dep set (latest stable at 2026-05-25): see implementation plan for full version pin list.
- Single binary, no daemon.

## Appendix A — Telemetry (deferred, NOT in 1.0)

If/when telemetry is added (target 1.1+):

- **Local-first.** All events written to `~/.local/state/llmenv/telemetry.db`
  (sqlite). No network egress by default. `llme stats` queries the DB.
- **Structured events as a typed Rust enum** → JSON Lines into sqlite, schema
  versioned. Examples: `ScopeActivated{scope_id, duration_ms, cache_hit}`,
  `BundleMaterialized{bundle_id, files_count, duration_ms}`,
  `SyncRun{result, duration_ms}`, `HookFired{shell, duration_ms}`.
- **Strict allowlist:** stable IDs only. Never paths, hostnames, project names,
  file contents, env values.
- **Sampling + retention caps** (default 30 days, hot-path events sampled
  1-in-N once baseline established).
- **`tracing` crate** as carrier; custom subscriber writes to sqlite; same
  pipeline feeds stderr logs.
- **Opt-in network export:** `llme stats export` produces a redacted JSON
  summary the user can hand-attach to bug reports. Never auto-uploaded.
- **Kill switches:** `LLME_TELEMETRY=0` env; `[settings] telemetry = false`
  in TOML.
